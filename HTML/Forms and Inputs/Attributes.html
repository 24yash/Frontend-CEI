HTML Form Attributes






Explanation of Form Attributes:

  • action: Specifies where to send form data when submitted (e.g., "/action_page.php")
  • method: Defines HTTP method for submitting form data (GET or POST)
  • target: Determines where to display the response after submission (_blank, _self, _parent, _top, or framename)
  • autocomplete: Enables/disables browser's autocomplete functionality
  • novalidate: Specifies that form input should not be validated on submission

Additional Form Attributes:

  • accept-charset: Specifies character encodings for form submission
  • enctype: Defines how form data should be encoded (for method="post")
  • name: Assigns a name to the form
  • rel: Specifies the relationship between linked resource and current document

Notes on GET vs POST methods:

GET:

  • Appends form data to URL in name/value pairs
  • Not secure for sensitive data (visible in URL)
  • Limited URL length (2048 characters)
  • Useful for bookmarkable form submissions
  • Suitable for non-secure data like search queries

POST:

  • Sends form data in the body of the HTTP request
  • More secure (data not visible in URL)
  • No size limitations
  • Cannot be bookmarked
  • Recommended for sensitive or personal information
